# STRIX Licensing

STRIX uses a dual-license model to balance open-source transparency with commercial protection.

## Existing Code (v0.1.x and earlier)

All code released as part of STRIX v0.1.x and earlier is licensed under the
**Apache License, Version 2.0**. See [LICENSE-APACHE](LICENSE-APACHE).

This includes all commits up to and including tag `v0.1.0`.

## New Code (v0.2.0 and later)

All new code, modules, and features added from v0.2.0 onward are licensed under the
**Business Source License 1.1** (BSL). See [LICENSE-BSL](LICENSE-BSL).

### What BSL means for you:

| Use Case | Allowed? |
|----------|----------|
| Reading, studying, and auditing the code | Yes |
| Non-commercial research and academic use | Yes |
| Personal and development/testing use | Yes |
| Contributing back to the project | Yes |
| Commercial production deployment | Requires commercial license |
| Military/defense operational deployment | Requires commercial license |

### Change Date

On **2030-04-13** (or four years after each version's release, whichever comes first),
the BSL-licensed code automatically converts to **Apache License 2.0**.

## Why dual license?

STRIX contains safety-critical algorithms for autonomous systems. The BSL model ensures:

1. **Transparency**: All code is publicly readable and auditable -- critical for safety-critical systems
2. **Academic freedom**: Researchers can use, study, and build upon STRIX freely
3. **Commercial sustainability**: Production deployments fund continued development
4. **Eventual open source**: All code becomes fully open source after 4 years
